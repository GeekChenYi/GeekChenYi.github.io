## Setup

`setup`是`vue3`新增的一个选项，它是组件内使用`Composition API`的入口

#### setup执行时机

`setup`的执行时机是在`beforeCreate`之前执行，可以通过下边的例子验证。

```javascript
export default defineComponent({
  beforeCreate() {
    console.log("----beforeCreate----");
  },
  created() {
    console.log("----created----");
  },
  setup() {
    console.log("----setup----");
  },
});
// 打印结果
// ----setup----
// ----beforeCreate----
// ----created----
```

### setup参数

使用`setup`时，它接收两个参数：

- props：组件传入的属性

- context

#### props

`setup`中接受的`props`是响应式的，当传入新的`props`时，会及时被更新。

由于`props`是响应式的，因此不可以使用ES6语法中的解构赋值，解构会消除它的响应式。

错误代码示例，这段代码会让`props`不再支持响应式：

```javascript
export default defineComponent({
  setup(props, context){
    const { name } = props
    console.log(name)
  }
})
```

那我们在开发中，想要使`props`解构的方式使用，还要保持props的响应式，有没有办法解决呢？

答案肯定是有的，需要使用到另一个Composition api `toRefs`

#### context

由于在`setup`中是不能访问`Vue2`中最常用的`this`对象，所以`content`中就提供了`this`最常用的三个属性：`attrs`、`slot`和`emit`，分别对应Vue2.x中的`$attr`属性、`slot`属性、`$emit`属性，并且这几个属性都是自动同步最新的值，所以我们每次使用拿到的都是最新值。

### ref

在vue3中，我们可以通过一个新的 `ref` 函数使任何响应式变量在任何地方起作用，一般用来定义基本类型的数据

```js
import { ref } from 'vue'
const counter = ref(0)
```

`ref` 接收参数并将其包裹在一个带有 `value` property 的对象中返回，然后可以使用该 `value`  property 访问或更改响应式变量的值： 

```js
import { ref } from 'vue'
const counter = ref(0)
console.log(counter) // { value: 0 }
console.log(counter.value) // 0
counter.value++
console.log(counter.value)// 1
```

### reactive

在 vue2.x 中， 定义数据都是在`data`中， 但是 Vue3.x 可以使用`reactive`和`ref`来进行数据定义。 那么`ref`和`reactive`他们有什么区别呢？分别什么时候使用呢？

`reacttive`用来代理一个对象，但是不能代理一个基本类型，而`ref`也是可以代理一个对象

```javascript
<div>
  <p>第{{ year }}年</p>
  <p>姓名：{{user.nickname}}</p>
  <p>年龄：{{user.age}}</p>
</div>

export default defineComponent({
  components: {Demo},
  setup() {
    const year = ref(0)
    const user = reactive({ nickname: 'xiaofan', age: 23})
    setInterval(() => {
      year.value++
      user.age++
    }, 1000)
    return {
     year,
     user
    }
  },
})
```

在上边的代码中，我们同样不能直接将`user`解构出来直接绑定到页面上，如果我们就想使用解构后的数据，解决办法还是和解构`props`是一样的，使用`toRefs`

### toRefs

`toRefs`用于将一个`reactive`对象转化为属性全部为`ref`对象的普通对象。

具体使用方式如下：

```js
const state = reactive({
    foo: 1,
    bar: 2
})
const stateAsRefs = toRefs(state)
// stateAsRefs的类型: { foo: Ref<number>, bar: Ref<number> }
state.foo++
console.log(stateAsRefs.foo.value) // 2

stateAsRefs.foo.value++
console.log(state.foo) // 3
```

```javascript
<div>
  <p>第{{ year }}年</p>
  <p>姓名：{{ nickname }}</p>
  <p>年龄：{{ age }}</p>
</div>

export default defineComponent({
  components: {Demo},
  setup() {
    const year = ref(0)
    const user = reactive({ nickname: 'xiaofan', age: 23})
    setInterval(() => {
      year.value++
      user.age++
    }, 1000)
    return {
     year,
     ...toRefs(user)
    }
  },
})
```

当从组合函数返回响应式对象时，`toRefs`非常有用

```js
function useFeatureX(){
    const state = reactive({
        foo: 1,
        bar: 2
    })
    return toRefs(state)
}
export defualt {
    setup(){
        const { foo, bar } = useFeatureX()
        return {
            foo,
            bar
        }
    }
}
```

`toRefs`只会为源对象中包含的所有property生成`ref`，如果要为指定的property创建`ref`，则应当使用`toRef`

### watch与watchEffect的用法

:::warning

watch 函数用来侦听特定的数据源，并在回调函数中执行副作用。默认情况是惰性的，也就是说仅在侦听的源数据变更时才执行回调。

如果不想让其成为惰性的值，设置immediate:true

:::

```javascript
watch(source, callback, [options])
```

参数说明：

- source: 可以支持 `string`,`Object`,`Function`,`Array`; 用于指定要侦听的响应式变量
- callback：执行的回调函数
- options：支持`deep`、`immediate`和`flush`选项

#### 侦听reactive定义的数据

```javascript
import { defineComponent, toRaw, ref, onMounted, onUnmounted, reactive, toRefs, watch } from 'vue'
setup() {
  const year = ref(0)
  const user = reactive({ nickname: 'xiaofan', age: 23})
  setInterval(() => {
    year.value++
    user.age++
  }, 1000)
  watch(() => user.age, (curAge, preAge) => {
    console.log("新值:", curAge, "老值:", preAge);
  })
  return {
   year,
   ...toRefs(user)
  }
}
```

#### 侦听ref定义的数据

```javascript
const year = ref(0)
setTimeout(() => {
  year.value++
}, 1000)
watch(year, (newVal, oldVal) => {
   console.log("新值:", newVal, "老值:", oldVal);
})
```

#### 侦听多个数据

上面两个例子中，我们分别使用了两个 watch, 当我们需要侦听多个数据源时， 可以进行合并， 同时侦听多个数据：

```javascript
watch([() => state.age, year], ([curAge, newVal], [preAge, oldVal]) => {
console.log("新值:", curAge, "老值:", preAge); console.log("新值:", newVal,
"老值:", oldVal); });

```

#### 侦听复杂的嵌套对象

```javascript
const state = reactive({
  room: {
    id: 100,
    attrs: {
      size: "140平方米",
      type: "三室两厅",
    },
  },
});
watch(
  () => state.room,
  (newType, oldType) => {
    console.log("新值:", newType, "老值:", oldType);
  },
  { deep: true }
);

```

如果不使用第三个参数`deep:true`， 是无法监听到数据变化的。 

#### stop停止监听

我们在组件中创建的`watch`监听，还在组件被销毁时自动停止。

如果在组件销毁之前我们想要停止掉某个监听，可以调用`watch()`函数的返回值，操作如下：

```javascript
const stopWatchRoom = watch(() => state.room, (newType, oldType) => {
    console.log("新值:", newType, "老值:", oldType);
}, {deep:true});

setTimeout(()=>{
    // 停止监听
    stopWatchRoom()
}, 3000)

```

### watchEffect

`watch`已经基本满足监听的需求了，为什么还要有`watchEffect`呢？

```javascript
import { defineComponent, ref, reactive, toRefs, watchEffect } from "vue";
export default defineComponent({
  setup() {
    const state = reactive({ nickname: "xiaofan", age: 20 });
    let year = ref(0)

    setInterval(() =>{
        state.age++
        year.value++
    },1000)

    watchEffect(() => {
        console.log(state);
        console.log(year);
      }
    );

    return {
        ...toRefs(state)
    }
  },
});

```

执行结果首先打印一次`state`和`year`值；然后每隔一秒，打印`state`和`year`值。从上面的代码可以看出， 并没有像`watch`一样需要先传入依赖，`watchEffect`会自动收集依赖, 只要指定一个回调函数。在组件初始化时， 会先执行一次来收集依赖， 然后当收集到的依赖中数据发生变化时， 就会再次执行回调函数。

对比总结：

- watchEffect 不需要手动传入依赖
- watchEffect 会先执行一次用来自动收集依赖
- watchEffect 无法获取到变化前的值， 只能获取变化后的值

### computed函数

```js
import { ref, computed, reactive } from 'vue'
const counter = ref(0)
const user = reactive({ nickname: 'xiaomi', age: 12 })

const twiceTheCounter = computed(() => counter.value * 2)
const u = computed(() => user.age + 12)

counter.value++
console.log(counter.value)// 1
console.log(twiceTheCounter.value) // 2
console.log(u.value) // 24
```

### 响应式API

#### readonly

接受一个对象 (响应式或纯对象) 或 [ref](https://v3.cn.vuejs.org/api/refs-api.html#ref) 并返回原始对象的只读代理。只读代理是深层的：任何被访问的嵌套 property 也是只读的。 

```js
const original = reactive({ count: 0 })
const copy = readonly(original)
watchEffect(() => {
    console.log(copy.count)
})
// 没问题
original.count++

// 变更副本将失败并导致控制台警告
copy.count++
```

#### isProxy

检查对象是否为`reactive`或`readonly`创建的`proxy`

#### isReactive

检查对象是否是由 `reactive` 创建的响应式代理。 

如果该代理是`readonly`创建的，但包裹了有`reactive`创建的另一个代理，它也会返回`true`

#### isReadonly

检查对象是否为`readonly`创建的只读代理

#### toRaw

返回 `reactive` 或 `readonly`代理的原始对象。 

### Refs

#### unref

如果参数是一个`ref`，则返回内部值，否则返回参数本身，这是`val = isRef(val) ? val.value : val`的语法糖函数

#### toRef

可以用来为源响应式对象上的某个 property 新创建一个 [`ref`](https://v3.cn.vuejs.org/api/refs-api.html#ref)。

```js
const state = reactive({
  foo: 1,
  bar: 2
})

const fooRef = toRef(state, 'foo')

fooRef.value++
console.log(state.foo) // 2
console.log(fooRef.value) // 2

state.foo++
console.log(state.foo) // 3
console.log(fooRef.value) // 3
```

当你要将`prop`的`ref`传递给复合函数时，`toRef`很有用：

```js
export default{
    setup(props){
        useSomeFeature(toRef(props, 'foo'))
    }
}
```

`toRefs`只会为源对象中包含的property生成`ref`，如果要为特定的property创建`ref`，则应当使用`toRef`

检查值是否为一个 ref 对象。
